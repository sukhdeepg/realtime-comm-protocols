WebSocket provides a persistent, bidirectional connection between client and server. Unlike HTTP, it maintains a single connection where both client and server can send messages at any time.

Server-Sent Events (SSE)

SSE allows the server to push updates to the client over a single HTTP connection. It's one-way (server to client only) but the browser automatically reconnects if the connection is lost.

Long Polling

Long polling is where the client makes a request and the server holds the request open until data is available. After receiving a response, the client immediately sends a new request. This simulates real-time updates using standard HTTP.
